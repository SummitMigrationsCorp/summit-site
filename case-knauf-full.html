Executive Summary

Knauf, one of Europe’s most recognized manufacturers, partnered with Summit Migrations to modernize its global file storage footprint. With over 400 on‑premises file servers spread across 20+ countries — including offline facilities without internet access — this engagement required precise planning, multilingual coordination, and innovative technical solutions. Through a structured, country‑by‑country approach, Summit delivered a seamless global migration with zero data loss and complete retirement of all legacy servers.

Client Overview

Knauf operates across Europe, Asia, and North America, with regional IT structures and multilingual data environments. Their file infrastructure consisted of hundreds of aging file servers distributed across headquarters, regional hubs, and remote manufacturing sites — some of which lacked basic internet access. Knauf required a global shift to Microsoft OneDrive to standardize collaboration, reduce IT overhead, and prepare for a cloud‑first future.

Challenges

Multilingual Environments

Folder structures, permissions, and stakeholders spanned over a dozen languages, requiring careful coordination without a multilingual team on-site.

Offline Facilities

Several plants had no internet connectivity, making traditional cloud migration tools unusable.

400+ File Servers

The distributed infrastructure demanded rigorous inventory management and phased execution.

Global Coordination

Approvals and IT stakeholders were dispersed across North America, Europe, and Asia, increasing communication complexity.

Spotlight: Solving the Offline Facility Challenge

Some remote plants had absolutely no internet access. To solve this, Summit deployed Microsoft Azure Data Box — a physical appliance used to securely transfer data to Azure. Data was copied onsite, the device shipped to Microsoft for ingestion, and the migrated dataset was integrated into the standard OneDrive pipeline. This rare solution ensured no facility was excluded.

Our Approach

1. Global Discovery & Master Inventory

We cataloged every file server, share, permission structure, and owner across all in‑scope countries, producing a unified global migration inventory.

2. Offline Location Remediation via Azure Data Box

Offline sites were onboarded using Data Box, enabling a fully integrated migration pipeline despite the absence of internet connectivity.

3. Phased Migration by Country

Migrations were executed in phases using Microsoft Migration Manager and ShareGate. A dedicated PM coordinated multilingual communications and stakeholder approvals.

4. Reconciliation & File Server Retirement

No server was retired without full validation: data fidelity, permission accuracy, end‑user access checks, and local IT sign‑off.

The Results

  • Complete retirement of 400+ on‑premises servers
  • Zero data loss incidents across the entire program
  • Offline facilities successfully migrated using Azure Data Box
  • Multilingual stakeholder coordination executed without issue
  • No timeline disruption during tooling transition (Migration Manager → ShareGate)

Key Takeaways

  • Even the most complex infrastructures can be migrated cleanly with the right methodology.
  • Offline or bandwidth‑limited sites require creative engineering solutions.
  • Global change succeeds when stakeholder alignment is handled proactively.
  • Rigorous reconciliation is the backbone of a zero‑loss migration.
