Executive Summary

Edelman Financial Engines, one of the largest independent RIA firms in the United States, engaged Summit Migrations as its dedicated M&A migration and M365 governance partner. With frequent acquisitions and strict SEC/FINRA oversight, the firm required flawless execution and trusted technical stewardship. Summit now manages every acquisition migration, all compliance data preparation, and ongoing security remediation across the tenant.

Client Overview

Edelman Financial Engines serves hundreds of thousands of clients nationwide and operates an active M&A strategy, acquiring independent advisory practices and integrating their Microsoft 365 tenants into a unified environment. Each acquisition introduces a new tenant, its own IT footprint, and sensitive financial records requiring precise handling and regulatory care.

Challenges

Zero‑Margin‑for‑Error Migrations

Financial data including tax records, investment documentation, and PII is governed by SEC and FINRA. Any misstep carries compliance exposure.

Record Center Requirements

All client data must land in a Salesforce‑integrated Record Center, requiring structured metadata preparation before ingestion.

Continuous M&A Activity

The firm acquires new advisory practices frequently, each requiring full tenant‑to‑tenant migration.

Governance Gaps Between Migrations

Summit discovered recurring security risks including excessive external sharing, orphaned permissions, and unmanaged admin access.

Spotlight: The M&A Migration Model

Every acquisition follows a high‑stakes pipeline. Summit performs full T2T migration — email, OneDrive, SharePoint Online, and file shares — then constructs the Meta File: a master spreadsheet mapping each financial document to its Financial Planner, District Office, Salesforce ID, and compliance record number. This enables compliant ingestion into the firm's Salesforce‑integrated Record Center.

Scope of Services

M&A Tenant‑to‑Tenant Migration

ShareGate‑driven, wave‑based migrations with full audit trails and validation.

Record Center Data Preparation

Metadata enrichment, ID reconciliation, and compliance packaging via Meta File.

Security & Governance Advisory

External sharing remediation, admin access cleanup, and ongoing tenant hardening.

M365 Platform Administration

SharePoint builds, permissions management, Power Automate solutions, and internal content migrations.

Featured Initiatives

External Sharing Audit

PowerShell scripts provided full tenant visibility, resulting in a firm‑wide lockdown of external sharing.

Site Collection Admin Cleanup

Removed former vendor/contractor admin access across the tenant, eliminating longstanding compliance risk.

Internal Home Folder Migration

Retired on‑prem home folders and standardized OneDrive for all advisors.

SharePoint Archiving Initiative

Implemented SharePoint Premium archiving to control storage costs and reduce clutter.

Migration Methodology

  1. Pre‑Migration Assessment – Full tenant inventory and mapping.
  2. Pilot Migration – Validate mappings and environmental nuances.
  3. Wave‑Based Migration – Structured departmental waves with delta syncs.
  4. Meta File Construction – Integrate with Salesforce identifiers for Record Center ingestion.
  5. Source Archiving & Closeout – Final compliance validation and data retirement.

The Results

  • Zero data loss across all acquisition migrations.
  • Record Center pipeline standardized and fully operational.
  • Tenant security posture significantly strengthened.
  • Legacy storage eliminated and storage spending optimized.

Key Takeaways

  • Regulated‑industry migrations demand airtight execution.
  • Compliance workflows such as Record Center ingestion must be engineered strategically.
  • Ongoing governance ensures long‑term security and operational consistency.
  • Summit's steady, multi‑year partnership reflects trust built on results.
